## Goal

Update resolved character profiles through a character-scoped tool that replaces the complete value of one profile field when new evidence changes the best current short characterization.

## Context

The scoped profile patch tool removed full-profile proposals and the reviewer call, but it only fills missing fields. Character profiles are short current characterizations rather than append-only fact stores. New evidence must be able to refine, weaken, correct, or contradict an existing non-empty field.

## Scope

- Give the profile updater only the resolved character name, current profile, and bounded new evidence paragraphs.
- Expose one `replace_profile_field` tool scoped to the resolved character through construction.
- Accept a complete replacement value, evidence pointers from the current input, and a short diagnostic reason.
- Replace exactly one profile field per valid tool call, including existing non-empty fields.
- Preserve unaffected fields by making no tool call for them.
- Keep technical validation for supported fields, non-empty values, allowed evidence pointers, and existing character lookup.
- Remove unused proposal and reviewer contracts from the Character Bible profile-update path.
- Use profile-update terminology in new logs and progress messages while keeping the existing workflow executor name if needed to limit the diff.

## Non-goals

- Do not change collection, extraction, identity resolution, aliases, evidence indexing, vector search, or split limits.
- Do not add semantic review, observation memory, or programmatic profile summarization.
- Do not expose `characterId` as a tool parameter or LLM input.
- Do not append profile text or merge additions.

## Plan

1. Replace the prompt input and system prompt with compact profile-update contracts.
2. Change the scoped tool to replace full field values and record diagnostic reasons.
3. Remove obsolete proposal and reviewer classes and update focused tests.
4. Run repository checks with manual local-LLM tests excluded.

## Pause points

- Stop if replacing a field requires changing persisted dossier storage.

## Acceptance criteria

- The LLM receives only `target.name`, `currentProfile`, and `newEvidence`.
- `replace_profile_field` receives no `characterId` and replaces exactly one complete profile field value.
- Existing non-empty profile fields can be replaced.
- Empty values and pointers outside the current evidence context are rejected without writes.
- A profile updater that calls no tools leaves the profile unchanged.
- Multiple tool calls can update multiple fields while preserving unaffected fields.
- Proposal and reviewer DTOs are not used by the workflow.
- Repo checks pass with manual/local LLM tests excluded.
