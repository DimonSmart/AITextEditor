## Goal

Let character bible generation run while the user navigates between Web UI pages, keep the editor read-only during the automatic operation, and make extraction batching configurable with paragraph overlap.

## Context

Character extraction currently processes sequential paragraph batches with no overlap. A character name at the end of one batch can be unavailable to pronoun or role references at the start of the next batch. The Bible Operations page also owns the running task directly, so leaving the page cancels the operation and hides the running progress.

## Scope

- Add configurable character extraction batching settings to persisted program settings.
- Expose batch paragraph count, byte limit, overlap paragraph count, and full-scan item limit on the Settings page.
- Use overlap when splitting paragraph batches for character extraction.
- Replace the hidden full-scan hard limit with a configured limit.
- Keep a scoped background character bible operation state that survives navigation within the current Blazor circuit.
- Make Book editing and manual Character Bible edits read-only while an automatic bible operation owns the workspace.
- Notify subscribed pages when workspace state or character dossiers change so pages can refresh while the operation continues.

## Non-goals

- Do not add durable operation history.
- Do not add a multi-operation queue.
- Do not add multi-user locking or cross-process coordination.
- Do not change the model extraction contract.
- Do not implement a separate companion bible path.

## Plan

1. Add settings model, validation, UI fields, and workflow wiring for extraction batching limits.
2. Add overlap-aware paragraph batch splitting and tests.
3. Add workspace automation lock/read-only state with an `IDisposable` lease and mutation guards.
4. Move operation ownership into a scoped background state service and keep the operations page as a view over that state.
5. Subscribe Character Bible and Book pages to workspace change notifications and disable manual edits while read-only.
6. Update focused tests and run repository checks.

## Pause points

- Stop if keeping the operation alive across navigation requires changing service lifetimes outside the current Blazor circuit.
- Stop if incremental character display requires a broad rewrite of the staged workflow commit model.

## Acceptance criteria

- Character extraction batches can overlap by a configured number of paragraphs.
- Settings persisted by the Web app include visible controls for extraction batch size, byte limit, overlap, and full-scan item limit.
- Starting character bible generation does not get cancelled by leaving the Bible Operations page.
- Returning to Bible Operations shows the same run and progress events.
- The Book editor and manual Character Bible actions are disabled while generation is running.
- The Character Bible page can refresh from workspace changes while generation runs or after it completes.
- Existing non-manual tests pass.

## Verification

Commands or manual checks:

```bash
dotnet build AiTextEditor.sln
dotnet test --filter "Category!=Manual"
```

## Outcome

Implementation keeps operation ownership in a scoped `CharacterBibleOperationState`, so navigation away from `/bible-operations` no longer disposes the running task. The operation runner acquires an `EditorWorkspaceState.BeginAutomation()` lease while the workflow runs, and Book / Character Bible manual edits check the workspace read-only state.

Character extraction batching now supports paragraph overlap through `CursorAgentLimits.BatchOverlapElements`, and Web settings persist the batch paragraph limit, byte limit, overlap, and full-scan item limit.

The existing staged workflow still commits dossiers at the workflow commit stage. The Character Bible page refreshes from workspace notifications while state changes and after the operation commits; it does not yet receive a committed dossier update after each extraction batch.
