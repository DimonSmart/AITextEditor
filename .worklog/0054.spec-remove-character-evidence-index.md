# 0054 SPEC Remove Character Bible evidence index

Type: spec

Replaces:
- none

Related:
- 0029
- 0034
- 0052
- 0053

## Goal

Remove the persisted resolved evidence index from `CharacterDossiers`.

## Context

`evidenceIndex` was a derived index from resolved candidates to source evidence. It becomes stale when book pointers, excerpts, paragraph splitting, or character merge/split decisions change. Current catalog generation does not need this persisted index: it creates and updates dossiers from current candidates, identity decisions, and the current run state.

This spec supersedes the `evidenceIndex` persistence requirements from 0052. Resolved evidence should be rebuilt from runtime book indexes or run artifacts when needed.

## Scope

- Remove `EvidenceIndex` from the `CharacterDossiers` model and saved Character Bible JSON.
- Stop adding resolved candidate evidence to persisted dossiers during identity resolution.
- Remove evidence-index normalization from `CharacterDossierService`.
- Rename the remaining suspect evidence DTO from `CharacterEvidenceIndexEntry` to `CharacterEvidenceReference`.
- Keep `suspectArchive` evidence as unresolved candidate material with only `pointer` and `excerpt`.
- Keep loading old JSON with an unknown `evidenceIndex` field as an ordinary ignored JSON property.

## Non-goals

- Do not add a replacement persisted evidence index.
- Do not add a vector index in this change.
- Do not change identity resolution behavior.
- Do not change candidate extraction behavior.
- Do not remove `suspectArchive` evidence.
- Do not add a complex migration for old Character Bible JSON.

## Plan

1. Remove the `EvidenceIndex` property and all resolved-evidence index builders, adders, and normalizers.
2. Rename suspect evidence storage to `CharacterEvidenceReference` and keep it free of `CharacterId`.
3. Update tests around catalog creation, suspect archive evidence, final JSON serialization, and old JSON loading.

## Pause points

- Stop if profile update depends on persisted `EvidenceIndex` instead of current candidates and run state.
- Stop if old JSON loading requires a custom compatibility reader beyond ignoring an unknown field.

## Acceptance criteria

- `CharacterDossiers` has no `EvidenceIndex` property.
- Final Character Bible JSON does not contain `evidenceIndex`.
- There is no `AddEvidenceIndexEntries`, `BuildEvidenceIndexEntries`, or `NormalizeEvidenceIndex` code.
- `CharacterEvidenceIndexEntry` is replaced by `CharacterEvidenceReference`.
- `SuspectArchiveEntry.Evidence` stores only `Pointer` and `Excerpt`.
- New and existing identity decisions still create/update dossiers, merge observed name forms, set unknown gender, and record resolver decisions.
- Profile update continues to use current candidates and run state for new evidence.
- Old JSON containing `evidenceIndex` loads and saves back without that field.

## Verification

Commands:

```bash
dotnet build
dotnet test --filter "Category!=Manual"
```

Manual check:

```text
Run a Character Bible scenario on a book and inspect that characters, suspectArchive, identityConflicts, auditTrail, and profile updates still work while evidenceIndex is absent from the saved JSON.
```

## Outcome

Implementation notes:

- Persisted resolved evidence was removed from the `CharacterDossiers` model, edit session, resolution applier, normalization, tests, and the saved sample JSON.
- Legacy JSON with `evidenceIndex` is handled by the existing `System.Text.Json` unknown-property behavior and saves back without the field.
- `suspectArchive` evidence now uses `CharacterEvidenceReference` and serializes only `pointer` and `excerpt`.

