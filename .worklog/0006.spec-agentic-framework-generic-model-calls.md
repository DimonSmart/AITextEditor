# 0006 SPEC Agentic Framework generic model calls

Type: spec

Replaces:
- 0005

Related:
- 0002
- 0003

## Goal

Remove the project-owned manual OpenAI HTTP/JSON model-call layer and route all production model calls through Microsoft Agent Framework generic typed invocations.

## Context

The Semantic Kernel removal initially introduced typed OpenAI-compatible clients that manually posted to `/v1/chat/completions`, supplied `response_format` JSON schema bodies, parsed the OpenAI response envelope, and deserialized `choices[0].message.content`.

That still leaves a manual HTTP/JSON model-call layer in the project. The current requirement is stricter: model calls should use Agent Framework generic calls such as `ChatClientAgent.RunAsync<T>`, letting the framework request and parse structured output.

## Scope

- Add the OpenAI-compatible provider package needed by the Agent Framework chat client path.
- Build an `IAgenticModelClient` boundary backed by `ChatClientAgent.RunAsync<T>`.
- Wire character extraction, cursor command selection, cursor finalization, and test LLM assertions through that generic typed boundary.
- Remove `OpenAiStructuredChatModelClient`, `OpenAiCharacterExtractionModelClient`, and all hand-written OpenAI chat completion request/response DTOs.
- Remove the unused cursor-agent JSON response parser and its compatibility behavior.
- Remove request-body JSON rewriting from the LLM logging handler.
- Keep deterministic post-response contract validation for domain invariants that are not fully expressed by the DTO type.

## Non-goals

- Do not add fallback parsing for malformed model output.
- Do not keep compatibility with the removed Semantic Kernel or manual HTTP clients.
- Do not add embeddings, vector search, global consolidation, or human review in this change.

## Acceptance criteria

- `src` no longer contains project-owned OpenAI chat completion request/envelope DTOs.
- Production model calls use `ChatClientAgent.RunAsync<T>`.
- Unit tests verify that the generic Agent Framework call requests structured JSON output.
- Character extraction still fails fast when the typed response violates domain contract rules.
- Repository build and non-manual tests pass.

## Outcome

Implemented the Agent Framework generic model-call path:

- `AgenticFrameworkModelClient` creates an OpenAI-compatible `IChatClient`, wraps it in `ChatClientAgent`, and calls `RunAsync<T>`.
- Character extraction and cursor-agent calls now pass `ChatMessage` collections and receive typed DTOs from the framework.
- The old manual HTTP clients and unused cursor JSON parser were removed.
- The LLM logging handler now logs request/response bodies only; it no longer mutates provider JSON.

Verification completed:

```bash

dotnet test tests/AiTextEditor.Tests/AiTextEditor.Tests.csproj --filter "Category!=Manual"
```
