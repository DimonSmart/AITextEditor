# 0015 SPEC Fixed MudBlazor shell navigation

Type: spec

Replaces:
- none

Related:
- 0007
- 0013

## Goal

Make the left application navigation a stable shell element that does not scroll with page content or move with horizontally overflowing pages.

Keep MudBlazor dialogs and overlays on their normal global layer above page content.

## Context

The Web UI now uses MudBlazor for the character catalog and related pages. The left navigation should follow the standard MudBlazor application shell shape: `MudLayout` contains the app navigation drawer and `MudMainContent` contains only the active page body.

Pages such as the Character Bible catalog may need their own vertical or horizontal scrolling, but that scrolling must remain inside the content area and must not move the left navigation.

## Scope

- Organize the main layout around MudBlazor shell components instead of page-level navigation scrolling.
- Configure the standard MudBlazor providers at the shell level.
- Keep the left navigation fixed relative to the viewport across pages.
- Let page content own its scrolling when a page naturally overflows.
- Remove page-specific z-index workarounds that compete with MudBlazor's built-in overlay layers.
- Keep the existing navigation destinations and page behavior unchanged.
- Prefer MudBlazor layout components and parameters over custom positioning when practical.

## Non-goals

- Do not redesign unrelated pages.
- Do not add responsive navigation features beyond what is needed for a stable shell.
- Do not change character bible storage, filtering, editing, or operation behavior.
- Do not add custom global overlay z-index rules unless MudBlazor's default layer model cannot work.
- Do not add compatibility paths for older layout implementations.

## Plan

1. Inspect the current Blazor layout, navigation menu, global CSS, and MudBlazor setup.
2. Replace or adjust the layout shell to use `MudLayout`, `MudDrawer`, and `MudMainContent` consistently.
3. Move overflow ownership to the page content area so horizontal page scrolling cannot include the menu.
4. Run repository checks and a local Web smoke check if the app can start.

## Pause points

- Stop if the current MudBlazor package version lacks the needed layout behavior.
- Stop if fixing the shell requires changing page-specific workflows outside the layout boundary.

## Acceptance criteria

- The left navigation remains fixed while page content scrolls vertically.
- Horizontal overflow in pages such as Character Bible affects only the content area, not the menu.
- The layout follows MudBlazor shell conventions with drawer navigation in the main layout.
- MudBlazor dialogs, including delete confirmation, render above character cards and other page content.
- Existing navigation links still work.
- Existing non-manual tests pass.

## Outcome

The shell now uses `MudLayout`, `MudAppBar`, a fixed persistent `MudDrawer`, and `MudMainContent`. Body scrolling is disabled; the page body scrolls inside `.app-content`, so horizontal or vertical overflow belongs to the active page area instead of the left navigation.

The shell sizing rules live in global `app.css` rather than isolated layout CSS because the `app-shell`, `app-main-content`, and `app-drawer` classes are applied to MudBlazor component root elements. Isolated CSS did not reach those generated root elements, causing `MudMainContent` to expand to the full character card list while the body stayed `overflow: hidden`.

`MudThemeProvider` was added next to the existing MudBlazor providers. This is important for overlay ordering because the MudBlazor dialog container uses theme CSS variables such as `--mud-zindex-dialog`. The character edit dialog no longer sets custom overlay `z-index` values or a custom `BackgroundClass`; it relies on MudBlazor's default dialog layer and keeps only local opacity/border styling.

Verification completed:

```bash
dotnet build AiTextEditor.sln
dotnet test --filter "Category!=Manual"
```

Local HTTP smoke check returned 200 for `/character-bible`, and the rendered HTML contains both MudBlazor layout markup and `--mud-zindex-dialog`. In-app browser verification with the sample character bible showed 11 character cards, `.app-content` as the scroll container, and wheel scrolling changed `.app-content.scrollTop` while the drawer stayed `position: fixed`. Delete confirmation also rendered above the cards with MudBlazor's default dialog z-index.
