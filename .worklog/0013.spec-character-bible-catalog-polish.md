# 0013 SPEC Character Bible catalog polish

Type: spec

Replaces:
- none

Related:
- 0012

## Goal

Make the Character Bible page read as a usable character catalog: clear cards, predictable edit/delete actions, useful filtering, and an opaque edit dialog that does not visually blend with the page behind it.

## Context

The JSON Character Bible UI already replaced the raw Markdown/YAML editor with cards and structured editing. External review found that the current page still looks too much like loose text blocks: character boundaries, hierarchy, aliases, facts, toolbar grouping, and dialog contrast need to be improved.

## Scope

- Make each character render as a visibly bounded card with an avatar initial, name, gender badge, description, alias chips, fact count, and edit/delete actions inside the card.
- Improve the toolbar layout with title/count separation, search, gender filtering, incomplete filtering, add, and save actions.
- Filter search over character name, aliases, description, and facts.
- Add a small incomplete state computed from missing/default description and missing facts.
- Keep create/edit/delete behavior backed by the existing structured dialog and dossier service.
- Fix the edit dialog styling so it is visually opaque and separated from the page background.

## Non-goals

- Do not change the character bible storage format.
- Do not change the persisted character model unless the existing fields are insufficient for UI rendering.
- Do not add relationship graphs, timelines, analytics, or a full Markdown editor.
- Do not add a read-only detail drawer in this change.
- Do not redesign unrelated pages or the left navigation.

## Plan

1. Inspect the current Character Bible page, dialog, shared styles, and MudBlazor setup.
2. Extract or improve the card rendering while keeping the page state simple.
3. Update toolbar filtering and count behavior.
4. Add focused styles for cards, chips, incomplete/selected states, and dialog opacity.
5. Run repository checks and a local browser smoke check if the Web app can start.

## Pause points

- Stop if fixing dialog opacity requires changing global MudBlazor provider setup across unrelated pages.
- Stop if the current UI library does not support the requested controls without a broad package or theme change.

## Acceptance criteria

- Every character appears inside a clear card with avatar, name, gender badge, description, aliases, facts, and local actions.
- Aliases appear as chips with empty and overflow states.
- Edit and delete actions are visibly part of the relevant card, and delete still asks for confirmation.
- Search covers name, aliases, description, and facts, case-insensitively for Russian text.
- Gender and incomplete filters affect the displayed cards and counter.
- The grid adapts from three-ish columns on wide screens to one column on narrow screens.
- The edit dialog is opaque enough to read and no longer visually blends with the background.
- Existing storage behavior remains unchanged.

## Verification

Commands or manual checks:

```bash
dotnet build AiTextEditor.sln
dotnet test --filter "Category!=Manual"
dotnet run --project src/AiTextEditor.Web
```

## Outcome

The catalog now labels empty fact sets as `No facts` instead of `0 facts` to avoid implying that extracted facts were lost. Current generated dossiers still have no facts because the character extraction contract contains name, gender, aliases, and description only; `CharacterBibleCharacterCandidate` and `ProfileAccumulator.ToDossier()` do not carry `CharacterFact` values yet. Extracting facts requires a separate pipeline change to the typed response schema, prompt, candidate contracts, merge path, and tests.
