## Goal

Make dossier patch proposals return the best current short character profile after considering new evidence, instead of returning phrases that are appended to existing profile fields.

## Context

The additions-only contract accumulates observations but cannot refine, weaken, correct, or replace an earlier profile characteristic. Character profiles should remain concise current descriptions rather than append-only journals.

## Scope

- Keep the compact patch prompt input: `target.name`, `currentProfile`, and `newEvidence[] { pointer, text }`.
- Replace the proposal output with required `status`, full four-field `profile`, and explanatory `changes[]`.
- Support `append` and `replace` change actions.
- Require `noUsefulChanges` proposals to return the normalized current profile and an empty changes list.
- Require `updated` proposals to change at least one profile field and return at least one evidence-backed change.
- Apply approved proposals by using the returned profile directly.
- Preserve pointer membership validation in the patch stage.
- Update the reviewer prompt and focused tests for the new contract.

## Non-goals

- Do not change identity resolution or candidate extraction.
- Do not add profile fields, observation memory, retrieval, or tools.
- Do not expose technical workflow DTOs in the patch prompt.
- Do not change dossier storage beyond applying the returned profile.

## Plan

1. Replace additions DTOs and validation with profile update DTOs.
2. Rewrite the patch and reviewer prompts for refinement-aware profile updates.
3. Apply approved proposal profiles directly and keep pointer membership checks.
4. Update focused tests and run repository checks.

## Pause points

- Stop if applying the full profile requires changing the persisted `CharacterDossier` format.

## Acceptance criteria

- Patch proposal prompt no longer requests additions.
- Proposal output contains full `profile` and `changes`.
- Model client validates required fields and status-dependent constraints.
- Approved updates can replace an existing psychological profile instead of appending to it.
- One-time actions can result in `noUsefulChanges`.
- Prompt input remains compact.
- Repository checks pass with manual tests excluded.
