# 0051 SPEC Embedding server selection

## Goal

Allow LLM operations and Character Bible embedding/vector search operations to use different configured AI servers.

## Context

The Settings page already lets the user select an AI server and model for LLM calls, and a separate embedding model for vector search. Vector search currently resolves its endpoint from the selected LLM server, which prevents using a different embedding server when embeddings are served by another Ollama/OpenAI-compatible host.

## Scope

- Add a persisted selected embedding server setting next to the selected embedding model.
- Show and edit the selected embedding server on the Settings page.
- Resolve Character Bible vector search embedding settings from the selected embedding server.
- Keep the selected LLM server and selected embedding server independent.
- Add focused tests for persistence, validation, and embedding client configuration.

## Non-goals

- Do not add provider-specific model listing or discovery.
- Do not add fallback from embedding server to LLM server.
- Do not add migrations or legacy settings adapters.
- Do not change per-call embedding generation signatures.

## Plan

1. Update `ProgramSettings` with selected embedding server state and validation.
2. Update Settings UI selection and server rename/delete behavior.
3. Route `CharacterVectorSearchServices` through the selected embedding server settings.
4. Add focused tests and run repository checks.

## Pause points

- Stop if vector search currently cannot resolve a server without changing embedding generation call signatures.

## Acceptance criteria

- Settings persists both selected LLM server and selected embedding server.
- The embedding server selector appears next to the embedding model field.
- Character vector search uses the selected embedding server endpoint and credentials.
- Missing selected embedding server produces a clear validation error.
- Repo checks pass with manual/local LLM tests excluded.
