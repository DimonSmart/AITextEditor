## Goal

Allow Character Bible users to read a complete character dossier directly in the card grid while keeping editing in the existing edit dialog.

## Context

Character cards are currently compact catalog entries. The full profile fields are only comfortable to inspect through the edit dialog, which makes editing feel like the primary read path. The page should support a read-only expanded state for one character at a time.

## Scope

- Add a compact and expanded display mode to `CharacterCard`.
- Let the character name and a small view/collapse action toggle expansion.
- Render one expanded card across the full card grid width.
- Show read-only full profile sections and aliases in expanded mode.
- Hide empty full-mode sections and show a short empty profile state when nothing is filled.
- Keep `Edit` and `Delete` behavior backed by the existing dialog and delete flow.
- Preserve existing search, gender, importance, and incomplete filters.

## Non-goals

- Do not add a read-only dialog, separate page, or drawer.
- Do not add editable controls to the expanded card.
- Do not change the JSON model, save logic, or character generation algorithm.
- Do not add relationship fields or compatibility for removed dossier schema fields.
- Do not redesign unrelated pages or navigation.

## Plan

1. Update page state so only one `ExpandedCharacterId` is tracked.
2. Update `CharacterCard` parameters and markup for compact and expanded modes.
3. Add focused card-grid and expanded-view CSS.
4. Run repository checks.

## Pause points

- Stop if the dossier model has no stable character id.
- Stop if expanding to full grid width requires replacing the page layout with a broad refactor.

## Acceptance criteria

- Character Bible cards render compact by default.
- Clicking a character name expands or collapses that card.
- Expanding another card collapses the previous one.
- The expanded card spans the full grid width.
- Expanded mode shows read-only profile sections and aliases.
- Empty expanded sections are hidden, with a short empty profile state when no sections are filled.
- Edit still opens the existing edit dialog.
- Delete keeps the existing confirmation and removal behavior.
- Filters and search continue to affect the visible cards.
- Saving from the edit dialog updates the expanded card content.

## Outcome

Implementation uses MudBlazor `MudGrid`/`MudItem` for the responsive card layout. Expanded cards switch their item width to `xs=12 sm=12 lg=12`; compact cards keep `xs=12 sm=6 lg=4`.

The current simplified dossier model has no `description` or relationship field, so the expanded read-only view renders the four current profile sections plus aliases and alias examples.
