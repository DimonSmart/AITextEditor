# 0038 SPEC Resolver evidence prompt contract

Type: spec

Replaces:
- none

Related:
- 0029
- 0037

## Goal

Make the character identity resolver prompt input self-contained for LLM reasoning by exposing one materialized `evidence` list and removing candidate-level `pointers` from the resolver prompt payload.

## Context

The identity resolver currently serializes a candidate projection with `candidate.pointers` and a separate text list. That shape looks like an internal DTO join and can make the model treat pointers without visible text as hidden evidence. The resolver should reason only from text actually present in the prompt.

## Scope

- Add explicit LLM-facing input records for character identity resolution.
- Serialize resolver candidate identity fields without `pointers`.
- Serialize all visible resolver evidence as `{ pointer, text }` items in a single `evidence` array.
- Fail before the LLM call if resolver evidence has a pointer without usable text.
- Update resolver prompt wording to refer to `evidence` and forbid reasoning from omitted paragraphs or `candidateId`.
- Add focused tests for multiple evidence pointers and absence of candidate `pointers` in JSON.

## Non-goals

- Do not change extraction output.
- Do not change vector search.
- Do not change resolver output.
- Do not change dossier patch or review prompt contracts except where tests need to stay aligned with resolver changes.

## Plan

1. Introduce resolver prompt input records and project `CharacterBibleCharacterCandidate` into them.
2. Replace resolver prompt JSON fields from `candidate.pointers` plus `paragraphs` to `candidate` plus `evidence`.
3. Add prompt-input diagnostics and fail-fast validation for unmaterialized evidence text.
4. Update prompt text and unit tests.

## Pause points

- Stop if resolver prompt input turns out to share a DTO with patch/review prompts.
- Stop if evidence is intentionally trimmed before the resolver prompt.
- Stop if the Agent Framework path requires the old candidate `pointers` field.

## Acceptance criteria

- Resolver prompt JSON does not contain `candidate.pointers`.
- Resolver prompt JSON contains `evidence` with pointer and text for every candidate evidence item.
- A candidate with multiple evidence pointers serializes all of them.
- Missing evidence text fails before the resolver model call.
- Resolver prompt instructions no longer ask the model to join pointers with separate context arrays.
