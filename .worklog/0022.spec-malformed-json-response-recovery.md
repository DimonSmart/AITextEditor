# 0022 SPEC Malformed JSON response recovery

Type: spec

Replaces:
- none

Related:
- 0003
- 0006
- 0011

## Goal

Recover valid structured responses when a model wraps JSON in extra text or Markdown fences, and keep character bible extraction running when one batch returns unrecoverable malformed JSON.

## Context

Generic Agent Framework typed calls currently retry `ChatClientAgent.RunAsync<TResponse>` on JSON parsing failures and finally throw the request contract error. In character bible extraction this means one malformed batch response can fail the whole workflow.

The agent project already references `DimonSmart.AiUtils`, which provides `JsonExtractor.ExtractJson(string?)` for extracting valid JSON fragments from surrounding text.

## Scope

- Add a small internal recovery helper for typed structured responses.
- Use `JsonExtractor` only to extract an already valid JSON fragment from raw model text.
- Add logging for malformed raw responses, recovery attempts, recovery success, and recovery failure.
- Add an Agent Framework raw/untyped response path if the typed path fails and raw text is available through public APIs.
- Keep the current typed retry behavior, with recovery attempted between failed attempts.
- Make character bible candidate extraction skip recoverable failed batches and continue later batches.
- Report skipped batch warnings through existing progress text.
- Add focused unit tests for JSON recovery and skipped extraction batches.

## Non-goals

- Do not repair truncated JSON by adding brackets, quotes, or fields.
- Do not call another LLM to repair JSON.
- Do not split failed batches into smaller batches.
- Do not change character extraction prompts or response DTO schema.
- Do not use reflection or internal Agent Framework types to get raw responses.

## Plan

1. Inspect the current Agent Framework model client API usage and available raw response APIs.
2. Add `StructuredJsonResponseRecovery` with direct deserialization first and `JsonExtractor.ExtractJson` fallback.
3. Wire recovery and structured logging into `AgenticFrameworkModelClient`.
4. Catch known recoverable extraction failures per batch in `CharacterBibleCandidateExtractor`.
5. Add tests for recovery cases and skipped-batch workflow behavior.

## Pause points

- Stop if raw response text is not available through public Agent Framework or `IChatClient` APIs and document that recovery is skipped for typed parse failures without raw text.

## Acceptance criteria

- JSON wrapped in prose or Markdown fences can be recovered and deserialized.
- Truncated JSON is not artificially repaired.
- Character bible extraction skips one failed batch and continues processing later batches.
- Progress text reports skipped batches and final warnings when any batch fails.
- Logs include raw response preview and recovery result for malformed responses.
- Existing happy-path character bible tests keep passing.
