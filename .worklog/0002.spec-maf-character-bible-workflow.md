# 0002 SPEC MAF character bible workflow

Type: spec

Replaces:
- none

Related:
- 0001

## Goal

Add a Microsoft Agent Framework based character bible workflow that can build or refresh character dossiers through an explicit graph pipeline.

## Context

The project already has Markdown linearization, cursor streams, Semantic Kernel tool plugins, and an in-memory character dossier generator. The target architecture now includes Microsoft Agent Framework migration in scope, so the next implementation should introduce a graph workflow without discarding existing deterministic services.

## Scope

- Add the Microsoft Agent Framework workflow package to the agent project.
- Add typed workflow request/result contracts for character bible generation.
- Add MAF executors for deterministic traversal/processing and dossier commit orchestration.
- Build a first graph workflow with `WorkflowBuilder`.
- Expose a service API that runs the MAF workflow and returns current `CharacterDossiers`.
- Wire the existing character dossier generator through the MAF workflow for the first implementation step.
- Add focused unit tests for workflow construction and execution with a scripted chat completion service.

## Non-goals

- Do not build full post-pass consolidation in this change.
- Do not persist the canonical character bible Markdown file yet.
- Do not add vector search or embeddings yet.
- Do not replace the existing Semantic Kernel command agent.
- Do not add human approval UI yet.

## Plan

1. Add `Microsoft.Agents.AI.Workflows` to `AiTextEditor.Agent`.
2. Introduce `CharacterBibleWorkflowRunner` and minimal typed contracts.
3. Implement one or two deterministic MAF executors that delegate to existing services.
4. Add tests that prove the workflow runs and updates dossiers.
5. Keep the existing `CharacterDossiersPlugin` behavior intact while making the workflow runner available for the next integration step.

## Pause points

- Stop if the current MAF package API differs enough from official docs that a compile-only adaptation would require broad rewrites.

## Acceptance criteria

- Character bible generation can run through a MAF workflow class.
- Existing character dossier generation behavior remains available.
- The implementation keeps a single write path through `CharacterDossierService`.
- Unit tests cover the new workflow runner.
- Repo checks pass.

## Verification

Commands or manual checks:

## Outcome

Implemented the first MAF migration step with `Microsoft.Agents.AI.Workflows` 1.4.0.

The workflow currently has one graph executor that delegates to the existing `CharacterDossiersGenerator`. This keeps the current extraction and write path stable while moving `generate_character_dossiers` and `refresh_character_dossiers` onto a MAF `WorkflowBuilder`/`InProcessExecution` path.

The package required `Microsoft.Extensions.Logging.*` 10.0.6, so the explicit logging package references were raised from 10.0.1 to 10.0.6.

Verification:

```bash
dotnet test --filter "Category!=Manual"
dotnet build AiTextEditor.sln
```

Follow-up remains: split extraction, matching, resolving, and committing into separate workflow executors after the domain contracts are introduced.
