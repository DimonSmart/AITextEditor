# 0031 SPEC LLM retry and log errors

Type: spec

Replaces:
- none

Related:
- 0006
- 0011
- 0030

## Goal

Make operation-log errors visually explicit and keep all production LLM agent calls behind one shared retry mechanism.

## Context

Malformed structured model responses are already retried, and raw malformed responses are copyable from the Character Bible operation log. The current retry loop lives inside the Agent Framework client itself, while the UI renders malformed-response diagnostics as ordinary progress rows with a prominent outlined copy button.

The log should make error diagnostics red and the raw-response copy action quieter. The LLM request retry behavior should be isolated in a named component so every agent path that calls the shared `IAgenticModelClient` uses the same retry behavior.

## Scope

- Add an explicit error flag for workflow progress and operation events.
- Mark malformed model-response diagnostics as error progress.
- Render operation-log error rows in red, including progress diagnostics.
- Render copyable raw-response actions as a quiet text/link-style action.
- Extract the LLM retry loop from `AgenticFrameworkModelClient` into a dedicated retry component.
- Keep cursor, extraction, dossier patch, reviewer, archive resolver, and split-candidate agents routed through the shared `IAgenticModelClient`.

## Non-goals

- Do not add JSON fragment extraction or malformed-response repair.
- Do not change model prompts or response schemas.
- Do not add a second model-call API for special-case agents.
- Do not persist raw model responses.
- Do not redesign the operation page layout.

## Plan

1. Add error metadata to workflow progress and operation events.
2. Update operation-log rendering and styles for error rows and quiet copy actions.
3. Move retry execution into a dedicated Agent model retry class.
4. Add focused tests for error forwarding, retry coverage, and UI/event behavior.

## Pause points

- Stop if model calls outside `IAgenticModelClient` are found in production code.

## Acceptance criteria

- Malformed model-response diagnostics are visibly red in the operation log.
- Copying malformed raw JSON is still available but no longer rendered as a prominent outlined button.
- `AgenticFrameworkModelClient` delegates retry attempts to a dedicated retry component.
- Production LLM agent clients still call through `IAgenticModelClient`.
- Non-manual repository checks pass.

## Outcome

Implementation notes:

- `AgenticModelRetryStrategy` owns structured response deserialization, malformed-response retries, and raw model request exception retries. `AgenticFrameworkModelClient` now only validates request shape and delegates through the shared strategy.
- Operation progress now carries `IsError`, so malformed responses, skipped failed batches, failed patch proposals, resolver fallbacks, and workflow-failed skip messages can render as explicit error rows without text matching in the UI.
- The raw-response copy action remains available but is rendered as a text-style link action.
