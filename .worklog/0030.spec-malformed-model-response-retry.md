# 0030 SPEC Malformed model response retry

Type: spec

## Goal

Handle malformed structured model responses by keeping copyable raw-response diagnostics and relying on ordinary retry attempts, without trying to recover JSON from partial or wrapped raw text.

## Context

The previous malformed-response path tried to recover JSON by passing raw model text through `JsonExtractor`. In practice this adds noisy failed-recovery events and does not improve Character Bible generation, while a normal retry can produce a valid structured response.

The operation log still needs to show when a model response could not be parsed and keep the full raw response copyable for diagnosis.

## Scope

- Deserialize the model response only as the complete structured JSON response.
- Report malformed response diagnostics with the full raw response available for copying.
- Retry malformed responses up to the existing structured response attempt limit.
- Report retry start and retry success diagnostics.
- Keep skipped-batch handling for character extraction batches that still fail after retries.
- Keep model response summary statistics focused on parse errors, retries, retry successes, and skipped batches.
- Remove `StructuredJsonResponseRecovery` and tests that validate JSON fragment extraction.

## Non-goals

- Do not extract JSON fragments from raw LLM responses.
- Do not repair truncated, wrapped, or schema-invalid JSON.
- Do not call another LLM to repair JSON.
- Do not change the character extraction JSON schema.
- Do not store raw model responses on disk.

## Plan

1. Replace structured response recovery with direct `System.Text.Json` deserialization of the full raw response.
2. Remove recovery diagnostic kinds, counters, UI labels, and tests.
3. Keep retry diagnostics and raw-response copy payloads.
4. Remove the now-unused `DimonSmart.AiUtils` package reference from the Agent project.

## Pause points

- Stop if another current model-call path still depends on JSON fragment extraction for production behavior.

## Acceptance criteria

- A model response wrapped in Markdown or prose is treated as malformed and retried.
- Operation progress no longer reports JsonExtractor recovery success or failure.
- Retry success remains visible in the operation log.
- Character extraction can still skip a batch after all retry attempts fail.
- Existing non-manual tests pass.
