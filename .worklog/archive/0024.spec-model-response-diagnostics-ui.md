# 0024 SPEC Model response diagnostics UI

Type: spec

Replaces:
- none

Related:
- 0011
- 0022

## Goal

Show malformed model response diagnostics in the character bible operation log, make the full raw response copyable, and include model response error statistics in operation results.

## Context

Agent Framework typed deserialization can fail before the recovered raw model text is visible in the Web UI. Server logs contain the exception and sometimes a raw preview, but the operation log only shows normal extraction progress. To diagnose whether the model returned valid JSON wrapped in prose or a truly malformed/truncated response, the user needs a short visible event plus a way to copy the full raw response.

## Scope

- Report malformed typed response events through the existing workflow progress path.
- Include a copyable full raw model response on progress events that have raw response text.
- Report recovery attempt outcome in the operation log when JSON extraction succeeds or fails.
- Keep retry events visible enough to understand whether retry helped.
- Add model response error statistics to the character bible workflow output and operation summary.
- Count skipped extraction batches and paragraphs in the same statistics object.

## Non-goals

- Do not store raw model responses on disk.
- Do not show raw response text inline in the operation log.
- Do not add another LLM repair step.
- Do not change the character extraction JSON schema.

## Plan

1. Add model-response diagnostics to `AgenticModelRequest`.
2. Bridge diagnostics from character extraction into `CharacterBibleWorkflowProgress`.
3. Add optional copy payload fields to operation events and render a copy button in the Bible Operations page.
4. Carry model response error statistics from extraction into the workflow output and UI summary.
5. Add focused tests for diagnostics, copy payload propagation, and statistics.

## Pause points

- Stop if Agent Framework does not expose full raw response text through public APIs in the fallback call.

## Acceptance criteria

- The operation log can show a short malformed response message with a copy button for the full raw response.
- Successful recovery and failed recovery/retry attempts are visible as progress events.
- Completed operation summary includes model response error statistics.
- Skipped extraction batches are counted in output statistics.
- Existing non-manual tests pass.

## Outcome

Implementation uses a raw-first model call path in `AgenticFrameworkModelClient`: it still sends the JSON schema through `ChatOptions.ResponseFormat`, but it reads `ChatResponse.Text` directly and deserializes through `StructuredJsonResponseRecovery`. This avoids first-chance `AgentResponse<T>.Result` JSON exceptions and makes the copyable diagnostic payload the actual raw model response from that attempt.

Diagnostics are bridged into character bible workflow progress and then into Web operation events. Model diagnostics remain visible in completed operation logs, and raw responses are kept only in memory as event copy payloads.
