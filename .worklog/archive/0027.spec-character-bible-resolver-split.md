## Goal

Split the current deterministic Character Bible resolver into explicit archive search, identity resolution, and commit plan building services without changing behavior.

## Context

The workflow now has separate extraction and dossier patch stages, but `CharacterBibleResolver` still combines archive indexing/search, existing/new/ambiguous identity decisions, dossier projection, alias/gender merge, importance scoring, and commit plan creation. The target architecture needs these responsibilities separated before adding an LLM identity resolver and suspect archive.

## Scope

- Add a read-only `ArchiveSearchService` that exposes deterministic exact/name/alias/normalized-key search results.
- Add deterministic identity resolution that converts a local candidate and archive hits into an `IdentityResolutionDecision`.
- Add `CommitPlanBuilder` that turns candidates and identity decisions into a `CharacterBibleCommitPlan`.
- Keep existing `Existing`, `New`, and `Ambiguous` behavior stable.
- Keep current dossier projection and importance behavior stable.
- Keep the workflow and public output shape stable.

## Non-goals

- Do not add `SuspectArchiveResolverAgent` yet.
- Do not add suspect archive persistence.
- Do not add `defer` or `identity_conflict` behavior beyond existing contracts.
- Do not add reviewer or split candidate branches.
- Do not change Character Bible storage contracts.

## Plan

1. Introduce archive hit and identity decision contracts.
2. Move read-only match/index logic from `CharacterBibleResolver` into `ArchiveSearchService`.
3. Move decision selection into deterministic identity resolution.
4. Move dossier projection and importance scoring into `CommitPlanBuilder`.
5. Leave `CharacterBibleResolver` as a narrow facade for current callers or remove it if all callers can use the builder directly.

## Pause points

- Stop if preserving exact existing matching behavior requires changing public dossier storage contracts.

## Acceptance criteria

- `CharacterBibleResolver` no longer owns archive search, identity decision, and commit plan projection in one class.
- `ArchiveSearchService` is read-only.
- `CommitPlanBuilder` does not call LLM services.
- Ambiguous candidates still do not create or merge dossiers.
- Existing and new candidate behavior remains covered by tests.
- Repo checks pass with manual/local LLM tests excluded by the standard filter.

## Outcome

Implemented the deterministic resolver split while preserving current behavior.

`CharacterBibleResolver` is now a narrow facade. It uses read-only `CharacterArchiveSearchService`, deterministic `DeterministicIdentityResolution`, and `CharacterBibleCommitPlanBuilder`. Dossier projection and importance updates moved into a separate projection index used by the commit plan builder.

One preservation detail is important: identity resolution remains sequential. Each candidate searches the current projected archive, so candidates created earlier in the same run can still be matched by later candidates, matching the old mutable resolver behavior.
