# 0010 SPEC AI server profiles

Type: spec

Replaces:
- none

Related:
- 0006
- 0009

## Goal

Allow the user to keep multiple OpenAI-compatible AI server settings, quickly select the active server, and select the active model name used by editor AI operations.

## Context

The Settings page currently stores one LLM connection. Local Ollama can expose an OpenAI-compatible endpoint, so the editor does not need provider-specific switching between OpenAI and Ollama. It needs OpenAI-compatible server profiles, an editor for those profiles, and separate persisted values for the currently selected server and currently selected model.

## Scope

- Replace the single editable LLM connection in the Settings UI with a server profile editor.
- Persist a collection of OpenAI-compatible server profiles.
- Persist the selected current server separately from the profile definitions.
- Persist the selected current model name separately from the profile definitions.
- Use the selected server profile and selected model when running AI operations.
- Keep environment variables only as initial defaults when no saved settings exist.
- Add focused tests for settings persistence and selected profile resolution.

## Non-goals

- Do not add provider-specific Ollama behavior.
- Do not add compatibility support for old settings file shapes.
- Do not fetch model lists from providers.
- Do not add secrets encryption, accounts, or sync.

## Plan

1. Inspect the current settings model, store, Settings page, and workflow configuration.
2. Add typed server profile settings and selected server/model state.
3. Update the Settings UI with profile create/edit/delete/select behavior.
4. Wire AI operations to resolve the selected profile and selected model.
5. Add focused tests and run repository checks.

## Pause points

- Stop if existing settings persistence cannot store a list of profiles without replacing the settings file contract.

## Acceptance criteria

- Settings contains an editor for multiple OpenAI-compatible server profiles.
- The current server selection is saved separately from server profile values.
- The current model name is saved separately and used for AI operations.
- AI operations fail with a clear UI-facing error when no selected server profile or model is configured.
- Existing non-manual tests still pass.

Implemented the profile-based settings path:

- `ProgramSettings` now stores OpenAI-compatible `AiServerSettings` profiles.
- The selected server name and selected model name are stored separately from profile values.
- The Settings page has profile add/edit/delete/select controls and a separate active model field.
- Character bible AI operations resolve the selected profile and selected model before creating the OpenAI-compatible client.
