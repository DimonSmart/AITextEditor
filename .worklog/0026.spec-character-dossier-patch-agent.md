## Goal

Restore Character Bible profile generation in a separate dossier patch stage after deterministic identity resolution.

## Context

Extraction now produces local character candidates with pointer-backed evidence and no profile content. This keeps candidate extraction focused, but generated dossiers temporarily have empty profile fields. The next architecture slice should add a dedicated LLM stage that proposes profile patches only for resolved `existing` and `new` candidates, leaving identity decisions and commit mechanics deterministic.

## Scope

- Add a `DossierPatchProposalAgent` model client with a Markdown system prompt resource.
- Add typed patch proposal contracts where `null` means "do not change this field".
- Run patch proposal after deterministic resolver decisions for `Existing` and `New` only.
- Apply approved-by-default patch proposals through `CharacterProfile.MergeMissing` semantics.
- Keep ambiguous/deferred candidates unchanged.
- Update focused tests for profile restoration and prompt shape.

## Non-goals

- Do not add `DossierConsistencyReviewerAgent` yet.
- Do not split `CharacterBibleResolver` into search, identity, and commit plan services yet.
- Do not add suspect archive, split proposal, or manual review persistence.
- Do not let the patch agent change identity, canonical name, gender, aliases, or evidence.
- Do not use empty strings as delete commands.

## Plan

1. Add patch proposal DTOs, model client, prompt builder, and embedded Markdown prompt.
2. Insert a patch stage between resolution and commit in the workflow.
3. Apply profile patches only to existing/new resolved dossiers and only for missing profile fields.
4. Update tests and verification.

## Pause points

- Stop if adding the patch stage requires changing public Character Bible storage contracts.

## Acceptance criteria

- Extraction remains profile-free.
- Profile text is generated by the patch agent stage after resolution.
- Existing non-empty profile fields are not overwritten by patch proposals.
- Patch proposals with `null` fields leave those fields unchanged.
- Ambiguous candidates do not call the patch agent.
- Repo checks pass with manual/local LLM tests excluded by the standard filter.

## Verification

## Outcome

Implemented a post-resolution dossier patch stage in the Character Bible workflow. The workflow now runs extraction, deterministic resolution, dossier profile patching, and commit as separate stages.

The patch agent has its own embedded Markdown prompt and typed JSON contract. It only runs for `Existing` and `New` decisions, merges missing profile fields, treats `null` as no change, and does not overwrite existing non-empty profile sections. Ambiguous candidates do not call the patch agent.
