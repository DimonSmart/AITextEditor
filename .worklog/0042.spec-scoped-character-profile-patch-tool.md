## Goal

Replace full-profile dossier patch proposals and mandatory LLM review with scoped, evidence-backed profile field updates for one resolved character at a time.

## Context

The full-profile proposal contract makes absent profile values appear as empty strings. The reviewer can then reject an otherwise useful update because `null -> ""` looks like an unsupported change. Patch application should be atomic at the profile-field level and validated by code before writing.

## Scope

- Give the patch agent the current character card, current nullable profile projection, and bounded new evidence paragraphs.
- Expose one `SetProfileField` tool scoped to the resolved character through construction.
- Validate non-empty concise values and evidence-pointer membership before applying a field update.
- Apply only missing fields. Return `NoOp` for equivalent text and `Conflict` for a different existing non-empty value.
- Let the patch agent call the tool zero or more times and finish without returning a full proposed profile.
- Remove the reviewer call from the ordinary patch path.
- Report patch-stage and per-tool-call statistics in the diagnostic log.

## Non-goals

- Do not add replace, append, or merge modes.
- Do not expose `characterId` as a tool parameter.
- Do not give the model full dossier write access.
- Do not add another reviewer.
- Do not rewrite extraction or identity resolution.

## Plan

1. Add scoped tool contracts, validation, and field-level session writes.
2. Replace the patch model contract and prompt with tool-driven completion.
3. Remove reviewer wiring from the active generator path and add patch statistics.
4. Update focused tests and run repository checks.

## Pause points

- Stop if field-level updates require changing persisted dossier storage.

## Acceptance criteria

- Patch flow does not invoke an LLM reviewer.
- Patch model output does not contain a full proposed profile.
- `SetProfileField` does not accept `characterId`; the scoped tool receives it through construction.
- Empty values and pointers outside the current evidence context are rejected without writes.
- Equivalent text is a no-op and different existing non-empty text is a conflict.
- A patch agent that calls no tools leaves the profile unchanged.
- Diagnostic `patch.done` includes processed-character, agent-call, tool-call, applied, no-op, rejected, conflict, and changed-field counts.

## Verification

Commands:

```bash
dotnet build
dotnet test --filter "Category!=Manual"
```

## Outcome

The scoped tool writes through the existing `CharacterDossierEditSession`, which already owns in-memory catalog mutation for the workflow. Missing profile values are normalized to `null`, so tool rejection and no-op paths cannot persist blank strings.

The earlier proposal and reviewer types remain compiled for isolated legacy contract tests, but production wiring and `patch_character_bible_dossiers` no longer construct or invoke them.
