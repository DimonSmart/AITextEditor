## Goal

Add an explicit program setting that controls the language used for Character Bible dossier profile field text generated by the profile update stage.

## Context

The profile update prompt input currently contains the update target, current profile, and new evidence, but no language setting. As a result, the model can mix Russian and English in generated profile fields when evidence, existing profile text, or prompt behavior drift across languages.

The language requirement belongs in the profile update input contract because it is part of the model decision context and should be visible in diagnostic logs.

## Scope

- Add `CharacterBibleDossierLanguage` to program settings with `Russian` as the default.
- Persist, clone, validate, and expose the setting in the Settings UI as a closed language select.
- Pass the validated setting into `CharacterProfileUpdatePromptBuilder`.
- Add `outputLanguage` to `CharacterProfileUpdatePromptInput`.
- Update profile update logging to include `outputLanguage` in the model input keys.
- Update the profile update system prompt so only profile field values are written in `outputLanguage`.
- Add focused tests for prompt input, default language, prompt rules, settings clone, and validation.

## Non-goals

- Do not change extraction, identity resolution, canonical name normalization, observed name forms, or evidence pointers.
- Do not translate names, observed forms, evidence excerpts, pointers, or quoted source phrases.
- Do not add language detection or regex validation to the profile update tool adapter.
- Do not add free-form prompt-like language instructions in the Settings UI.
- Do not change the persisted Character Bible JSON schema beyond generated profile field values.

## Plan

1. Add and validate the program setting.
2. Wire the setting through workflow validation and the web workflow client.
3. Extend the profile update prompt input and builder.
4. Update the profile update prompt and diagnostic logging.
5. Add focused tests and run repository checks.

## Pause points

- Stop if adding the setting requires changing existing saved settings through a migration or compatibility loader.

## Acceptance criteria

- The Settings page has a Character Bible dossier language select with Russian, English, and Spanish.
- Old settings without the language field use `Russian`.
- Profile update user prompt JSON contains `outputLanguage`.
- Profile update debug input logging lists `outputLanguage`.
- Profile update prompt rules scope language to profile field values only.
- Extraction, resolution, name normalization, observed forms, evidence pointers, and the tool adapter language validation remain unchanged.
- Non-manual repository checks pass.
