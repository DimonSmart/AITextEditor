# 0009 SPEC Program settings tab

Type: spec

Replaces:
- none

Related:
- 0007
- 0008

## Goal

Add a dedicated Settings tab where the user can configure program and LLM server connection values needed by the UI, and persist those settings across application restarts.

## Context

The Bible Operations page currently fails when required LLM configuration is missing, and the user has to set environment variables outside the application. The UI should expose the required connection settings directly so normal operation can be configured from the app.

## Scope

- Add a Settings navigation item and page to the Blazor app.
- Store LLM connection settings in a local application settings file.
- Load persisted settings on application startup.
- Allow the user to edit and save model, base URL, API key, optional basic auth credentials, SSL ignore flag, request body logging flag, and timeout.
- Use the saved settings when running character bible operations.
- Keep environment variables as initial defaults only when no saved setting exists.
- Add focused tests for settings persistence and workflow configuration.

## Non-goals

- Do not add user accounts, profiles, encryption, or cloud sync.
- Do not add compatibility shims beyond current environment variable names.
- Do not change the model-call protocol or workflow contracts.
- Do not implement operation history persistence.

## Plan

1. Inspect the Web project configuration flow and current workflow client.
2. Add a typed settings model and file-backed store.
3. Add a Settings page and navigation item.
4. Wire workflow execution to the saved settings.
5. Add focused tests and run repo checks.

## Pause points

- Stop if the current Web app cannot write a local settings file under the application data path.

## Acceptance criteria

- The app has a Settings tab in the main navigation.
- Settings entered in the UI survive restarting the app.
- Character bible operations use the saved LLM/server settings.
- Missing required settings produce a clear UI-facing error.
- Existing non-manual tests still pass.
