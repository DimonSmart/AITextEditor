# 0047 SPEC Profile update tool-only stage

Type: spec

Replaces:
- none

Related:
- 0030
- 0031
- 0046

## Goal

Make Character Bible profile update a tool-only stage where successful tool calls, or their absence, determine the group result instead of a structured final model response.

## Context

The profile update agent applies profile field replacements through tools. After those tools run, the current Agent Framework typed completion path still expects a structured JSON final response. Natural language final text such as `No updates needed`, an empty response, or `The profile has been updated` is then treated as malformed and retried even though the profile update work may already be complete.

For this stage the final text is decorative. A model call that completes without transport or model failure and produces no tool calls means no profile change was needed.

## Scope

- Remove the required typed response contract from the profile update model call.
- Ignore empty, plain-text, and non-JSON final responses for profile update.
- Determine each profile update group result from successfully applied tool calls and real model/tool failures.
- Treat zero applied tool calls after a successful model call as `NoChange`.
- Keep retry behavior for transport/model failures and tool execution or validation failures.
- Remove malformed-final-response warnings and retry attempts from the profile update stage.
- Optionally log ignored final response text at debug level.

## Non-goals

- Do not change the profile update tool arguments.
- Do not introduce a profile patch accumulator.
- Do not delay tool application until the end of the agent run.
- Do not change duplicate or repeated tool call semantics.
- Do not change structured response retry behavior for other LLM stages.
- Do not change the final character bible JSON schema.

## Plan

1. Add a non-typed agent model execution path for tool-only calls while preserving the shared retry mechanism.
2. Move profile update to the tool-only path and remove `CharacterProfileUpdateCompletion` from its public contract.
3. Keep profile update group status based on applied tool call counts and exceptions.
4. Add focused tests for plain-text final response, empty final response, no tool calls, and tool validation failure.

## Pause points

- Stop if the Agent Framework cannot execute tool calls without a typed final response through the shared model client boundary.

## Acceptance criteria

- A valid profile update tool call followed by plain text final output finishes as updated without malformed-response warning or retry.
- A valid profile update tool call followed by empty final output finishes as updated without malformed-response warning or retry.
- A successful model call with no profile update tool calls finishes as no change.
- Invalid tool arguments or tool validation failures still fail or retry according to the existing retry policy.
- Other structured LLM stages continue to report and retry malformed typed responses.
