# 0014 SPEC Remembered book folder loading

Type: spec

Replaces:
- none

Related:
- 0008
- 0012

## Goal

Make book loading depend on a known book folder path, remember the last loaded book path in program config, and allow repeat launches to reload the same book and companion bibles with one Load action.

## Context

The editor can currently load book content directly, including paths that are not tied to the app's persisted book location. The intended workflow is folder-based: the user points the app at the book location once, then later opens the program and presses Load to restore that same book and its companion files. Character bible loading must remain derived from the current book path.

## Scope

- Remove the direct book upload/load path that does not establish a book file path.
- Persist the last successfully loaded book path in the program configuration.
- On startup, initialize the book loader from the persisted path.
- Let the Load action use the persisted/current book path when no new path is selected.
- Continue loading the derived character bible companion file whenever a book is loaded.

## Non-goals

- Do not add multi-book workspaces.
- Do not add compatibility fallback paths for old book or bible locations.
- Do not add a separate configurable character bible path.
- Do not change the character bible JSON schema.
- Do not redesign unrelated settings or operation pages.

## Plan

1. Inspect the current Web book loading UI, workspace state, program settings store, and tests.
2. Add a persisted last book path setting using the existing settings store pattern.
3. Change the Book UI so Load requires a selected or remembered path and no longer accepts upload-only content.
4. Ensure workspace loading still calls the companion character bible store for the derived path.
5. Update focused tests and run repository checks.

## Pause points

- Stop if the existing settings store cannot safely persist the path without changing the settings file format broadly.
- Stop if the current browser/file picker API cannot provide a real book path and a desktop-side folder picker must be introduced first.

## Acceptance criteria

- A book cannot be loaded into the workspace through an upload-only path with no book file path.
- After a successful book load, the full book file path is stored in program config.
- After restarting the app, pressing Load uses the remembered book path when it still exists.
- Loading a book also loads the derived character bible companion file.
- Character bible path remains derived from the book file path, not stored separately.
- Existing non-manual tests pass.
