# 0023 SPEC Character bible overlap byte limit

Type: spec

Replaces:
- none

Related:
- 0003
- 0009
- 0016

## Goal

Limit character bible extraction overlap by both paragraph count and optional byte size.

## Context

Character extraction batches already respect both paragraph count and byte size for the main batch. Overlap currently uses only a paragraph count, so a few large paragraphs can make the next request much larger than expected or crowd out useful fresh text if overlap is treated as part of the main block.

## Scope

- Add a persisted overlap byte limit next to overlap paragraph count.
- Expose the overlap byte limit on the Settings page.
- Split paragraphs so the main batch and overlap are calculated separately.
- Use up to the configured overlap paragraph count, but keep only the largest suffix whose UTF-8 text size does not exceed the configured overlap byte limit.
- Keep at least one overlap paragraph when paragraph overlap is enabled and any overlap candidate exists, even when that paragraph exceeds the byte limit.
- Treat an overlap byte limit of `0` as unlimited by byte size.
- Preserve no-overlap behavior when overlap paragraph count is `0`.

## Non-goals

- Do not change the model extraction JSON contract.
- Do not change full-scan limits or character resolution behavior.
- Do not add compatibility aliases for older settings names.

## Plan

1. Add `OverlapMaxBytes` to extraction limits and Web settings.
2. Update paragraph batching overlap calculation to use a separate byte-limited suffix selection.
3. Add focused tests for overlap byte limiting and settings persistence/validation.
4. Add the new field to the Settings UI.

## Pause points

- Stop if preserving the existing forward progress behavior requires broad batching rewrites outside `CharacterBibleParagraphBatcher`.

## Acceptance criteria

- Overlap uses no more than the configured paragraph count.
- When overlap byte limit is greater than zero, overlap uses the maximum suffix that fits the byte limit.
- At least one paragraph is overlapped when paragraph overlap is enabled and overlap candidates exist.
- Main batch sizing remains independent from overlap sizing.
- Settings persistence, workflow validation, and UI include the overlap byte limit.
