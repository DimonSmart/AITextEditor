## Goal

Improve generated character dossier descriptions by giving the dossier patch stage full pointer-expanded evidence context instead of only short extraction excerpts.

## Context

The current pipeline correctly keeps character extraction narrow: it identifies local character candidates, aliases, pointers, and short evidence excerpts. This preserves stage boundaries, but the dossier patch agent receives only those short excerpts. Many excerpts are anchors such as `said X`, `asked X`, or a brief name mention. They often omit the actual dialogue, action, and surrounding paragraph content needed to infer speech style, behavior, competence, and personality.

The extractor should remain an identity/evidence anchor finder, not a profile writer. Richer profile synthesis should happen after identity resolution by expanding candidate evidence pointers into nearby document context in deterministic code.

## Scope

- Keep the extraction output contract profile-free.
- Keep extraction evidence as pointer-backed anchor excerpts.
- Add a code layer that expands candidate evidence pointers into context for the dossier patch stage.
- Include the current paragraph and a small nearby paragraph window when building patch input.
- Preserve anchor excerpt and pointer evidence so audit/review paths remain traceable.
- Prefer grouping patch input by resolved character where practical, so one patch call can see multiple evidence contexts for the same dossier.
- Treat grouped evidence as new evidence for the current run, not as a replacement for incremental dossier updates.
- Let the patch agent compare new evidence with the existing dossier and return no changes when no meaningful addition is present.
- Treat non-null profile patch fields as additive text, not as replacement text for existing non-empty fields.
- Split grouped evidence into bounded patch batches so one resolved dossier can be updated through several incremental patch calls when context would otherwise become too large.
- Keep agent prompts in separate Markdown resources.
- Update focused tests around prompt payload/context expansion and richer patch input.

## Non-goals

- Do not add LLM tool-calling for paragraph lookup in this slice.
- Do not move profile generation back into extraction.
- Do not ask extraction to choose long profile-worthy excerpts.
- Do not add embeddings, vector search, or semantic retrieval.
- Do not auto-overwrite existing non-empty profile fields.
- Do not build manual review UI.

## Plan

1. Add a document-backed evidence context expander keyed by semantic pointer.
2. Extend the dossier patch prompt input so candidate evidence includes anchor excerpts plus expanded context.
3. Adjust dossier patch orchestration to use expanded context, and group resolved evidence for the same dossier where this does not change identity behavior.
4. Tighten patch/reviewer prompts so they use context for the target character while preserving pointer-backed evidence.
5. Add tests for multi-sentence paragraphs and dialogue split across neighboring paragraphs.

## Pause points

- Stop if context expansion requires changing public Character Bible storage contracts.
- Stop if grouping patch input by dossier would require changing identity decisions.

## Acceptance criteria

- Patch agent input contains pointer, anchor excerpt, current paragraph text, focused text, and nearby paragraph context.
- A candidate whose evidence is only `X said:` can still give the patch stage access to the following dialogue paragraph.
- Existing profile fields are preserved and can receive additive new details from new evidence.
- New evidence that does not add meaningful dossier information can produce `no_useful_changes`.
- Patch prompt payloads are bounded by a code limit so grouped evidence does not grow without limit.
- Existing evidence index and audit trail continue to store pointer-backed evidence.
- Extraction prompt still does not request profile text.
- Repo checks pass with manual/local LLM tests excluded by the standard filter.

## Outcome

Implemented pointer-expanded dossier patch context without changing the extraction contract.

The patch stage now expands each candidate evidence pointer to:

- the anchor excerpt;
- the full current paragraph;
- focused text around the anchor when it can be located;
- one previous and one next non-heading document item.

Patch proposals are now grouped by resolved dossier, so multiple local candidates for the same character are sent to one dossier patch prompt instead of filling profile fields from isolated candidate snippets. The reviewer receives the same expanded evidence contexts.

The grouped prompt still represents only current-run evidence. The existing dossier is passed alongside it, and profile field patches are now additive: `null` means no new information, while a non-empty field is appended when it is not already represented in the existing field. This keeps incremental updates from being replaced by grouped evidence synthesis.

The grouped evidence is split into bounded patch batches before calling the patch agent. The current implementation limits a patch call to 8 candidate evidence groups or about 12 KB of expanded context; an oversized single evidence group is sent alone. Batches for the same dossier run sequentially, so later batches see the dossier after earlier additive patches.
