# 0011 SPEC Character bible progress events

Type: spec

Replaces:
- none

Related:
- 0003
- 0007

## Goal

Show detailed progress events for character bible operations so the UI reflects the actual workflow stages instead of only showing start and finish.

## Context

The staged character bible workflow already collects text, extracts candidates, resolves them, and commits dossiers. The operation UI currently exposes too little progress, making a completed run look opaque even when useful intermediate counts are available.

## Scope

- Emit progress events when the current document is loaded.
- Emit progress events before and after paragraph collection.
- Emit progress events before and after candidate extraction.
- Emit progress events before and after candidate resolution.
- Emit progress events before and after dossier commit / bible update.
- Include useful counts in progress messages when the stage result has them.
- Keep the event stream deterministic and readable for both full generation and changed-pointer refresh operations.

## Non-goals

- Do not change character extraction, matching, ambiguity, or commit behavior.
- Do not add a new UI view.
- Do not persist progress history outside the existing operation event stream.
- Do not add compatibility layers for old event wording.

## Plan

1. Locate the web operation runner and workflow client boundary.
2. Add a progress callback or event bridge that can report workflow stage messages.
3. Emit progress messages at the character bible workflow stages.
4. Add focused tests for the expected progress event sequence.

## Pause points

- Stop if detailed progress requires replacing the current workflow execution API.

## Acceptance criteria

- A character bible operation shows more than the generic loading/running messages.
- Progress includes reading/collecting text, candidate extraction, resolution, and bible update stages.
- Counts are shown for paragraphs, candidates, decisions, ambiguous decisions, and dossiers when available.
- Existing workflow result behavior remains unchanged.
- Repo checks pass.
