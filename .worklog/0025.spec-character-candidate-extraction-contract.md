## Goal

Narrow the Character Bible extraction step so it only extracts local character candidates with pointer-backed evidence and does not create dossier profile content.

## Context

The current Character Bible flow still asks the extraction LLM to identify characters and write structured profile fields in one pass. That makes the extractor responsible for both local evidence collection and dossier authoring, which obscures later identity resolution and patching responsibilities.

The target architecture separates these concerns: extraction produces candidates and evidence, code validates and normalizes them, resolution decides existing/new/ambiguous, and a later patch agent writes dossier changes.

## Scope

- Remove profile data from the extraction response contract.
- Add pointer-backed evidence to extracted characters and aliases.
- Rewrite the Character Bible extraction prompt as a candidate extraction prompt stored in a Markdown resource.
- Keep the current paragraph collection, batching, overlap, resolver, and commit path working.
- For this first step, allow committed dossiers to have empty profiles until a later patch agent restores profile generation in the correct stage.
- Update focused tests for the new extraction contract and prompt shape.

## Non-goals

- Do not add `DossierPatchProposalAgent` in this step.
- Do not split `CharacterBibleResolver` yet.
- Do not add suspect archive persistence or conflict branches.
- Do not introduce an LLM identity resolver.
- Do not add compatibility shims for the old profile-bearing extraction contract.

## Plan

1. Change extraction DTOs and validation so candidates carry `Evidence[]` and alias evidence instead of `Profile`.
2. Update `CharacterExtractionPromptBuilder` and its Markdown prompt resource for candidate-only extraction.
3. Update mapping/resolution code to preserve existing/new behavior while committing empty profiles from extracted candidates.
4. Update tests and JSON recovery fixtures that assert the old extraction profile shape.

## Pause points

- Stop if removing profile from extraction requires changing public Character Bible storage contracts.

## Acceptance criteria

- The extraction LLM output schema no longer contains profile fields.
- Extracted characters and aliases require evidence with a pointer and excerpt.
- The workflow still builds and commits character dossiers through the existing resolver and committer.
- Existing profile generation is intentionally absent from extraction and represented as empty dossier profile content for now.
- Repo checks pass or any remaining failures are reported with exact scope.

## Outcome

Implemented the first architecture slice: extraction now returns candidates with pointer-backed evidence and no profile object. The existing resolver and committer remain in place, so newly created or refreshed dossiers keep profile fields empty unless they already had manual profile content.
