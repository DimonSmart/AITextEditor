## Goal

Make the Character Bible agent code easier to navigate by separating workflow, extraction, prompt, and resolution responsibilities while preserving current behavior.

## Context

The Character Bible generator uses its own workflow and model agents, but the current implementation keeps workflow executors, prompt text, extraction request building, and deterministic dossier resolution close together. This makes the agent code harder to review and violates the principle of least surprise for class names and namespaces.

## Scope

- Move Character Bible code into explicit `AiTextEditor.Agent.CharacterBible` namespace groups.
- Extract the character extraction system prompt into an embedded Markdown resource.
- Add a prompt builder that loads the embedded prompt and builds the JSON user message.
- Split generator responsibilities into focused extraction, batching, and resolution services.
- Move MAF workflow executors out of the runner file.
- Replace Character Bible use of `CursorAgentLimits` with `CharacterBibleExtractionLimits`.
- Update Web and tests for the new namespaces and limits.

## Non-goals

- Do not change Character Bible extraction, resolution, persistence, progress wording, or UI behavior.
- Do not refactor CursorAgent prompt constants in this change.
- Do not add compatibility aliases for old namespaces or limit types.
- Do not add runtime-editable prompt files.

## Plan

1. Create the new Character Bible namespace/folder layout and embedded prompt resource.
2. Introduce `CharacterExtractionPromptBuilder` and `CharacterBibleExtractionLimits`.
3. Extract text batching, model extraction, and resolver logic from `CharacterDossiersGenerator`.
4. Move workflow executors to workflow-specific files.
5. Update callers, tests, and settings validation.

## Pause points

- Stop if preserving current behavior requires changing public Character Bible workflow result shape.

## Acceptance criteria

- Character Bible prompt text is no longer hard-coded in `CharacterDossiersGenerator`.
- Character Bible code no longer depends on `CursorAgentLimits`.
- Public Character Bible workflow surface stays focused on request, output, progress, and decisions.
- Existing behavior and progress messages remain unchanged.
- Prompt builder tests cover resource loading and JSON user prompt shape.

## Outcome

Implemented the structure refactor with the planned namespace split:

- public Character Bible surface moved under `AiTextEditor.Agent.CharacterBible`;
- extraction contracts, prompt builder, and prompt resource moved under `AiTextEditor.Agent.CharacterBible.Extraction`;
- workflow executors moved under `AiTextEditor.Agent.CharacterBible.Workflow`;
- deterministic resolution/index logic moved under `AiTextEditor.Agent.CharacterBible.Resolution`;
- Character Bible settings now use `CharacterBibleExtractionLimits` instead of `CursorAgentLimits`.

The extraction system prompt is now an embedded Markdown resource loaded by `CharacterExtractionPromptBuilder`. The builder fails fast when the resource is missing or empty and still builds the same JSON user prompt shape with `task` and `paragraphs`.
