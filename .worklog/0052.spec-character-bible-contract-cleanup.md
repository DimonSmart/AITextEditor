## Goal

Clean up Character Bible profile-update contracts, evidence indexing, and run metrics so diagnostics and LLM inputs match the current pipeline behavior.

## Context

The Character Bible pipeline can defer title-only or otherwise unresolved candidates into `suspectArchive`. That conservative behavior is correct, but current diagnostics and persisted evidence can make unresolved cases look like ordinary ambiguity or resolved evidence. The profile update input also carries repeated paragraph snippets, and profile-field length and semantic-field rules are easier for the model to discover after tool rejection than before generation.

## Scope

- Add separate deferred and suspect counters to Character Bible workflow statistics and diagnostics.
- Keep `ambiguousCount` for true ambiguous or merge-conflict decisions only.
- Persist `evidenceIndex` with only resolved `characterId` evidence.
- Deduplicate persisted `evidenceIndex` entries by `(characterId, pointer)`.
- Keep unresolved evidence available through `suspectArchive`.
- Compact profile-update `newEvidence` to pointer, required focused text, and optional short before/after context.
- Add profile field length guidance to the profile-update prompt and tool schema.
- Raise the hard profile field value limit enough that normal 500-character boundary values are not rejected.
- Tighten profile field semantic guidance so event retelling, competence, psychology, appearance, and speech are not mixed.

## Non-goals

- Do not merge title-only suspect candidates automatically.
- Do not change extraction window overlap behavior.
- Do not change the final Character Bible JSON schema except removing unresolved entries from `evidenceIndex`.
- Do not add a separate `suspectEvidenceIndex` unless unresolved evidence cannot stay only in `suspectArchive`.
- Do not add compatibility readers or legacy fallback paths.

## Plan

1. Update resolver/workflow statistics and progress/log output with deferred and suspect counts.
2. Filter and deduplicate evidence before persisted Character Bible output is written.
3. Replace expanded profile-update evidence input with the compact evidence DTO.
4. Update profile-update prompt and tool descriptions for length and field semantics.
5. Add focused tests for counts, evidence filtering/deduplication, compact input, and profile value length validation.

## Pause points

- Stop if removing unresolved entries from `evidenceIndex` would require downstream storage compatibility code.
- Stop if `suspectArchive` does not retain enough unresolved evidence to satisfy diagnostics.

## Acceptance criteria

- A deferred candidate produces `deferredCount > 0`.
- `suspectCount` reports entries added to `suspectArchive`.
- `ambiguousCount=0` no longer implies there were no unresolved deferred cases.
- Persisted `evidenceIndex` contains no `characterId: null` entries.
- Persisted `evidenceIndex` contains no duplicate `(characterId, pointer)` entries.
- Profile update LLM input no longer repeats anchor excerpt, full paragraph, and focused text for the same evidence.
- Normal profile values slightly above 500 characters are not rejected by the hard validator.
- The profile-update prompt and tool schema clearly ask for concise values and semantic field separation.
- Title-only suspect candidates remain in `suspectArchive`.

## Outcome

Implementation notes:

- `suspectCount` reports suspect archive entries added during the current workflow run, not the total archive size.
- Profile update evidence now serializes as `pointer`, `focusedText`, optional `contextBefore`, and optional `contextAfter`. The deterministic expander still names nearby paragraphs `previous` and `next`; the prompt builder maps those to before/after.
- The hard profile field value limit is 1000 characters while the prompt and tool schema target concise values at 500 characters or less.
