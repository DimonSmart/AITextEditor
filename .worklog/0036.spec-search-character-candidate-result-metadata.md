## Goal

Make `search_characters` explicitly expose vector search results as retrieval candidates for comparison, not confirmed character identity matches.

## Context

The identity resolver uses vector search over the current in-memory Character Bible archive. In small archives, vector search can return non-empty but unrelated results because it returns the closest available entries. The tool contract and detailed run log must make that limitation visible to the agent and diagnostics.

## Scope

- Update the `search_characters` tool and adapter descriptions to describe retrieval candidates, small-archive unrelated results, and vector similarity scores.
- Return a wrapper result with query, effective limit, archive size, returned count, explanatory note, and ranked hits.
- Add rank to each returned hit.
- Update resolver/search call sites and tests for the new result contract.
- Add archive size and returned count to detailed search log lines.

## Non-goals

- Do not add C# identity heuristics, thresholds, score interpretation, or automatic merge logic.
- Do not preserve the old array-only tool result contract.
- Do not change the vector search scoring algorithm.
- Do not add compatibility overloads or fallback search paths.

## Plan

1. Update contracts and DTOs for `ICharacterArchiveSearchTool`.
2. Update `CharacterArchiveSearchToolAdapter` mapping, descriptions, and detailed logging.
3. Update the tool registration metadata and tests.
4. Run repository checks with manual/local LLM tests excluded by local instructions.

## Pause points

- Stop if the new wrapper result cannot be represented through the existing tool factory metadata.

## Acceptance criteria

- `ICharacterArchiveSearchTool.SearchCharactersAsync` returns `CharacterArchiveSearchResult`, not `IReadOnlyList`.
- Tool descriptions state that results are retrieval candidates, small archives may return unrelated entries, and score is vector similarity rather than identity confidence.
- Parameter descriptions discourage generic query words and names from previous hits.
- Result contains query, limit, archive size, returned count, note, and hits.
- Each hit contains rank.
- Detailed log shows archive size and returned count.
